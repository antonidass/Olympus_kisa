# BOGI Flow Promptr — расширение Chrome

Помощник для ручной генерации картинок и видео в Google Flow:

- side panel с выбором сценария и списком промптов из `content/<миф>/prompts/{images,video}.md`
- кнопка **Copy** возле каждого промпта (кладёт в буфер → ты вставляешь Ctrl+V в Flow)
- авто-копирование следующего промпта когда ты жмёшь **Generate** в Flow
- перехват zip-архива при **Download Project** → автоматически вызывает
  `automation/distribute_images.py --execute` и раскладывает картинки в
  `images/review_images/scene_NN/vN.jpg`

Расширение не имитирует клики и не печатает сам — все действия делает
живой пользователь, поэтому Flow ничего не палит.

## Установка

1. Запустить webapp (он отдаёт расширению список сценариев и промпты):
   ```
   webapp\run.bat
   ```
2. Открыть `chrome://extensions`, включить **Developer mode** (правый верхний угол).
3. Нажать **Load unpacked** → выбрать папку `extension/`.
4. Закрепить иконку «BOGI Flow Promptr» на тулбаре. Клик по иконке открывает side panel.

## Использование

1. Открыть [labs.google/fx](https://labs.google/fx/...) или нужную страницу Flow.
2. Кликнуть по иконке расширения — справа откроется панель.
3. Выбрать сценарий и тип промптов (Картинки / Видео).
4. Первый промпт уже в буфере. Кликнуть в textarea Flow → Ctrl+V → Generate.
5. После клика Generate side panel автоматически переключается на следующую
   сцену и кладёт её промпт в буфер.
6. Когда все сцены прогнаны — три точки в углу Flow → **Download Project**.
   Скачанный zip автоматически уезжает в `distribute_images.py`, картинки
   раскладываются по сценам, ты переходишь в webapp выбирать варианты.

## Известные ограничения

- Эвристика на «Generate»: ловим клики по элементам, чей текст или
  `aria-label` похожи на `Generate / Create / Сгенерировать / Создать / Go`.
  Если Flow обновит интерфейс — возможно, придётся подкрутить TRIGGERS
  в [content.js](content.js).
- Перехват zip: фильтр на `labs.google / aisandbox / storage.googleapis.com`
  + `.zip`. Если архив качается с другого хоста — просто запусти
  distribute вручную из терминала.
- Sidepanel должен быть открыт в момент скачивания zip, иначе сообщение
  не дойдёт до UI (background-воркер сохранит pending в `storage.local`,
  но distribute стартует только когда sidepanel снова откроют — так
  безопаснее, чтобы случайный zip из Flow не уехал в чужой сценарий).
