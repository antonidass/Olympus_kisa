# Кисы Олимпа — пайплайн генерации видео

Полный пайплайн создания коротких анимированных роликов по греческим мифам для канала «Кисы Олимпа»: сценарий → промпты → картинки → видео → озвучка → монтаж.

## Структура проекта

```
BOGI AI/
├── content/                          # Контент по роликам (один ролик = одна папка)
│   ├── Икар и Дедал/                 # Имя папки = название мифа с заглавных букв и пробелами
│   │   ├── prompts/                  # Промпты для генерации
│   │   │   ├── images.md             #   картинки (Google Flow / Nano Banana)
│   │   │   ├── video.md              #   видео (Veo / LTX)
│   │   │   └── voiceover.md          #   текст озвучки
│   │   ├── voiceover/
│   │   │   └── audio/                # Сгенерированные mp3 (не в git)
│   │   ├── images/                   # Сгенерированные картинки (не в git)
│   │   │   └── scene_01/             #   по одной папке на сцену, 1-4 варианта v1.jpg … v4.jpg
│   │   ├── video/                    # Сгенерированные клипы (не в git)
│   │   ├── music/                    # Фоновая музыка (не в git)
│   │   └── final/                    # Готовый результат (не в git)
│   │       └── Икар и Дедал.mp4      #   Имя файла = имя папки
│   ├── Сизифов Труд/                 # Новый сценарий — та же структура
│   └── Ящик Пандоры/
├── automation/                       # Скрипты автоматизации
│   ├── imagefx_runner.py             #   Playwright-раннер для Google Flow (картинки)
│   ├── video_runner.py               #   Playwright-раннер для Google Flow (видео Veo)
│   ├── elevenlabs_runner.py          #   TTS через ElevenLabs
│   ├── flow_projects.py              #   Общий резолвер Flow-проектов (ID хранится в json)
│   ├── flow_projects.json            #   Маппинг «имя сценария → flow_id» (обновляется при первом запуске)
│   ├── dump_media.py                 #   Дамп медиафайлов
│   ├── inspect_flow.py               #   Инспекция пайплайна
│   └── requirements.txt
├── pyCapCut/                         # Сборка CapCut-драфтов из Python (монтаж)
│   ├── template_theseus.py           #   Эталонные параметры монтажа (зум, шрифты, переходы, громкости)
│   ├── build_<миф>.py                #   Базовая раскладка таймлайна
│   ├── enrich_<миф>.py               #   Зум, переходы, эффекты, громкости, SFX
│   └── karaoke_<миф>.py              #   Караоке-субтитры через whisper
├── assets/                           # Общие SFX и музыка для всех мифов
│   ├── audio/WHOOSH.mp3              #   Whoosh-SFX на стыках сцен
│   └── music/                        #   Базовые фоновые треки
└── CONTEXT.md                        # Контекст проекта (правила, стиль, персонажи)
```

### Именование папок сценариев

Папка сценария внутри `content/` называется точно как миф: **с заглавных букв, слова через пробел, без подчёркиваний и транслита**. То же имя используется для финального видеофайла.

- ✅ `content/Икар и Дедал/final/Икар и Дедал.mp4`
- ✅ `content/Сизифов Труд/final/Сизифов Труд.mp4`
- ❌ `content/икар_и_дедал/` (старый формат, больше не используем)

## Установка

```bash
# Автоматизация (Python)
cd "BOGI AI"
pip install -r automation/requirements.txt
python -m playwright install chromium

# pyCapCut (Python) — монтаж в CapCut-драфт
pip install -r pyCapCut/requirements.txt
```

Также нужен **ffmpeg** в PATH (используется webapp для склейки превью озвучки и cosyvoice_runner для конвертации). Скачать с [ffmpeg.org](https://www.ffmpeg.org/download.html#build-windows), распаковать и добавить `ffmpeg.exe` в системный PATH.

## Создание сценария через UI

В webapp в шапке списка мифов есть кнопка **«+ новый»**. Клик → модалка с полем имени → бэкенд `POST /api/scenarios/create` создаёт всю папочную структуру и три заготовки промптов:

```
content/<Название мифа>/
├── prompts/
│   ├── voiceover.md        # шаблон: интро + кликбейтный хук + основной текст
│   ├── images.md           # шаблон с правилами канала и Сценой 1
│   └── video.md            # шаблон с обязательным негативом
├── voiceover/
│   ├── audio/              # пусто — заполнится при генерации TTS
│   └── texts/              # пусто — сюда кладутся sentence_NNN.txt
├── images/                 # пусто — сюда кладутся scene_NN/vN.jpg
├── video/                  # пусто — сюда кладутся scene_NN_M.mp4
├── music/                  # пусто — фоновый трек кладётся вручную
└── final/                  # пусто — сюда CapCut экспортирует финал
```

После этого миф появляется в хабе со статусом «wip» (текста ещё нет). Дальше — обычный пайплайн: пишется текст в `voiceover.md`, разбивается на `voiceover/texts/sentence_NNN.txt`, генерируется озвучка и т.д.

**Имя папки = точное название мифа на русском, с заглавных букв, через пробел, без подчёркиваний.** То же имя используется для финального видеофайла.

Программный аналог (без UI) — `python webapp/app.py` запустить и руками POST'нуть, либо просто `mkdir`-ом, но через UI быстрее и заведомо без ошибок в структуре.

## Пайплайн создания ролика

1. Предложить 5 сценариев. Я выбираю один.
2. В webapp нажать **«+ новый»** и ввести точное название мифа — раскатятся все папки и шаблоны промптов (см. раздел «Создание сценария через UI» выше). Альтернатива — создать руками.
3. Далее пишется сценарий по выбранному мифу и вместе с ударениями отправляется мне на проверку. **Структура voiceover.md жёсткая:** `# Заголовок` → `<Имя>. Миф за минуту.` → **кликбейтный хук в 1–2 предложения** (удерживает зрителя в первые 3 секунды, эталон — Мидас) → основной текст истории. Подробности и шаблоны хуков — в `CONTEXT.md`, раздел «Кликбейтный хук после интро».
4. После правок текст попадает в `content/<Название мифа>/prompts/voiceover.md`
5. Далее надо разбить текст на предложения и положить в папку voiceover/texts (как например для сценария Сизифов Труд)
6. Далее открывается веб приложение для ревью озвучки. Я нажимаю на кнопку сгенерировать озвучку для текущего сценария. Жду. Выбираю озвучки. Если надо перегенерирую.
7. Далее пишутся промпты картинок (`prompts/images.md`), я даю апрув на эти промпты и если все ок то запускается `automation/imagefx_runner.py content/<Название мифа>/prompts/images.md` (при первом запуске спросит Flow-project ID и сохранит его в `flow_projects.json`). Результат — `content/<Название мифа>/images/review_images/scene_NN/vN.jpg`.

   **Перед запуском раннера:**
   - **Debug-Chrome должен быть запущен на порту 9222.** Двойной клик по `automation/launch_chrome_debug.bat` — откроется отдельный Chrome с профилем `%LOCALAPPDATA%\BogiAiChromeDebug`. В первый раз залогиниться в Google. Окно держать открытым на всё время генерации.
   - **Формат `images.md`:** парсер ищет блоки `## Сцена N` с одной строкой `**Промпт:** ...` внутри. Один шот = один блок `## Сцена`. Если у одного предложения несколько визуальных шотов — это несколько последовательных `## Сцена`-блоков (например `## Сцена 2 (sent_002 — шот 1)` и `## Сцена 3 (sent_002 — шот 2)`). Маппинг `sentence ↔ scene_NN` держать в HTML-комментарии в начале файла, как в Мидасе и Тесее.
   - **Уникальный subject-маркер в начале каждого промпта:** первые 3–4 слова каждого `**Промпт:**` — короткая английская фраза-действие, уникальная между всеми сценами файла (`persephone gathering spring flowers`, `demeter torch night search`, `hades scooping persephone meadow`). Google Flow автоматически берёт первые слова в имя экспортированного файла — уникальные префиксы дают однозначное файл ↔ сцена при **ручной** генерации в Flow (минуя раннер). Без этого правила в архиве получается каша из одинаково названных файлов на разные сцены. Проверка: `grep '^\*\*Промпт:\*\* ' images.md | sed -E 's/^\*\*Промпт:\*\* ([^,]+),.*/\1/' | sort | uniq -c | sort -rn` — все строки должны быть с `1`. Подробности — в `CONTEXT.md` раздел «Уникальный subject-маркер в начале каждого промпта».

   **Альтернатива — ручная генерация в Flow + `distribute_images.py`:**
   Если предпочитаешь работать в Flow руками (видеть варианты в UI, перегенерить отдельные сцены, редактировать промпт на лету), можно сгенерить все картинки вручную, скачать всё одним «Download all» и разложить через `automation/distribute_images.py`:

   ```bash
   # 1. Сухой прогон — план раскладки + подсказки по позиции для спорных префиксов
   python automation/distribute_images.py "content/<миф>/images/<archive>.zip"

   # 2. Скопировать готовую --map команду из вывода, проверить визуально, добавить --execute
   python automation/distribute_images.py "content/<миф>/images/<archive>.zip" --execute \
     --map "withered_wheat_field=15" --map "demeter_in_frozen=27" ...
   ```

   Как скрипт сопоставляет файлы со сценами:
   - **`✓` строгий префикс** — имя файла начинается с тех же слов что маркер сцены (Flow обычно так и делает). Это требует чтобы первое слово маркера было уникальным — см. правило выше.
   - **`~` Flow обрезал маркер** — иногда Flow съедает 1-е слово (`meadow earth chasm bursting` → `Earth_chasm_bursting`). Substring-matcher это закрывает.
   - **`?` fuzzy** (`--fuzzy`) — пересечение слов как множеств. Используется когда в Flow редактировался промпт (`Hades_abducting_Persephone` ≠ `hades scooping persephone meadow`). **Ненадёжно — обязательно сверить визуально перед `--execute`.**
   - **Подсказка по позиции в архиве** — для несопоставленных префиксов скрипт смотрит на их порядок в zip-архиве (Flow сохраняет порядок генерации) и предлагает сцену между ближайшими «якорями» уже сопоставленных. Если генеришь сцены строго по порядку — работает почти всегда.
   - **`--map "prefix=scene_num"`** — ручной override для тех случаев когда автомат не справился. Можно повторять флаг сколько нужно.

   Скрипт безопасен: dry-run по умолчанию (без `--execute` ничего не двигает), останавливается если есть несопоставленные файлы (не делает частичную раскладку), пишет каждую операцию `mv` в лог. Распакованный `_unpacked/` удаляется автоматически после успешной раскладки.

8. Далее пишутся промпты для видео (`prompts/video.md`) по изображениям которые прошли мое ревью. Для примера можно посмотреть как это было у Мидаса или предыдущего сценария. То же правило **уникального subject-маркера** в первых 3–4 словах каждого `**Промпт:**` применяется и здесь — на случай ручной генерации видео.
9. **Видео** — Я запускаю через веб приложение генерацию и апрувлю все сцены. `automation/video_runner.py content/<Название мифа>/prompts/video.md`. Результат — `content/<Название мифа>/video/`

10. **Монтаж** — Сборка идёт через pyCapCut в **три шага** в строгом порядке. **Эталон монтажа канала с 2026-04-27 — живой CapCut-драфт «Тесей и Минотавр»** (раньше был Мидас, но Тесей доведён вручную до канонического стиля). Все эталонные значения зафиксированы в **`pyCapCut/template_theseus.py`** — `enrich_<миф>.py` и `karaoke_<миф>.py` для будущих мифов **обязаны импортировать оттуда**, не дублировать константы. Утилита `pyCapCut/_inspect_theseus_live.py` печатает свежие значения из живого драфта — запускать после ручных правок чтобы перепроверить и обновить `template_theseus.py`. **Не ориентироваться на `build_midas.py` как на эталон стилей** — там устаревшие настройки.

   **Шаг 10.1 — Базовый драфт (`build_<миф>.py`):**
   - Скопировать `pyCapCut/scene_structure_theseus.py` → `pyCapCut/scene_structure_<миф>.py`. Перебить `SCENE_TEXTS` (короткие подписи на запас — `karaoke_*.py` их потом спрячет), список `SCENES` с маппингом `sentence_NNN_vX.mp3 ↔ scene_NN_v1.mp4` (из `prompts/video.md`). Несколько шотов в сцене = несколько `VideoShot` внутри одной `Scene`.
   - Скопировать `pyCapCut/build_theseus.py` → `pyCapCut/build_<миф>.py`. Поменять импорт на новый `scene_structure_<миф>`, `DEFAULT_PROJECT_NAME` (точное русское имя мифа), путь к папке контента и текст интро в спецкейсе сабтрека.
   - Запуск: `cd pyCapCut && PYTHONIOENCODING=utf-8 python build_<миф>.py`. Без `PYTHONIOENCODING=utf-8` Windows-консоль cp1251 падает на ⚠/→ после успешной обработки.
   - Драфт появляется в `%LOCALAPPDATA%\CapCut\User Data\Projects\com.lveditor.draft\<Название мифа>` с базовыми «плавными» переходами и неправильными громкостями. Это нормально — `enrich_*.py` всё перебьёт по эталону Тесея.

   **Шаг 10.2 — Обогащение переходами/эффектами/громкостями/зумом (`enrich_<миф>.py`):**
   - Скопировать `pyCapCut/enrich_theseus.py` → `pyCapCut/enrich_<миф>.py`. Поменять:
     - `THESEUS_DIR/THESEUS_FILE` → `<МИФ>_DIR/<МИФ>_FILE`
     - `SCENE_LAYOUT` — `(sid, число_шотов)` для каждой сцены, сумма должна совпасть с числом mp4 в `content/<миф>/video/`
     - `PLAN` — карта `(sid_конца_сцены, effect_id, длительность_сек, имя)` под драматургию мифа. Effect_id брать из `template_theseus.TRANSITIONS_LIBRARY` (там 17 переходов с длительностями 0.47–1.47s, выбранных вручную в живом Тесее). Утилита `_inspect_theseus_live.py` печатает свежий список.
     - `EFFECT_PLAN` — `template_theseus.VIDEO_EFFECTS_LIBRARY`: «Ожоги на пленке» на сцене 01, «Финальный круг» на финальную.
   - **CapCut должен быть закрыт** (включая трей).
   - Запуск: `PYTHONIOENCODING=utf-8 python pyCapCut/enrich_<миф>.py`.
   - Что делает (значения из `template_theseus.py`): применяет **зум 1.04 + transform.y −0.04** ко всем видео-сегментам (убирает водяной знак Veo), чистит старые переходы, выставляет громкости (`voice=1.0, video=0.34, music=0.1954, sfx_whoosh=1.0`), кладёт fade_out 2.9s на музыку, расставляет переходы по PLAN и whoosh-SFX на каждом стыке сцен (3 файла чередуются: `WHOOSH.mp3` × `Title Swoosh` × `Wind noise suitable for scene change`).
   - **Идемпотентно** — повторный запуск чистит свои предыдущие правки.

   **Шаг 10.3 — Караоке-субтитры (`karaoke_<миф>.py`):**
   - Скопировать `pyCapCut/karaoke_theseus.py` → `pyCapCut/karaoke_<миф>.py`. Поменять `THESEUS_NAME` → новое имя мифа, `CACHE_FILE` (`_karaoke_cache_<миф>.json`).
   - Подправить `build_intro_steps()` для накопительного интро («<Миф>. Миф за минуту.»). Шрифты и размеры — из `template_theseus.py`: интро на **Forum-Regular size 20 bold**, основные караоке-субтитры на **Philosopher size 12 bold**. Логика накапливающихся шагов на примере Тесея: «ТЕСЕЙ» → «ТЕСЕЙ И МИНОТАВР» → «+\nМИФ» → «+ЗА МИНУТУ». Тайминги — на старте слова, открывающего новый блок.
   - **CapCut должен быть закрыт** (включая трей).
   - Запуск из CosyVoice venv (там whisper + ffmpeg от CapCut):
     ```bash
     PYTHONIOENCODING=utf-8 ./external/CosyVoice/.venv_cosyvoice/Scripts/python.exe pyCapCut/karaoke_<миф>.py
     # повторно: добавить --no-transcribe чтобы не пересчитывать whisper
     ```
   - Что делает: транскрибирует mp3 через whisper-medium с word_timestamps, выравнивает с эталонным текстом из `voiceover/texts/sentence_NNN.txt`, удаляет статичные субтитры из шага 10.1 (кроме интро-плейсхолдера), добавляет две новые text-track: `intro` (Forum 20 bold, центр кадра, накопительные шаги) и `karaoke` (Philosopher 12 bold, КАПС, белый с чёрной обводкой).
   - Кэш транскрипций — в `pyCapCut/_karaoke_cache_<миф>.json`. Повторный запуск без перегенерации озвучки — мгновенный.

   **Что НЕЛЬЗЯ забыть** (на этом погорели первые проходы):
   - **Без `enrich_<миф>.py`** в драфте останутся базовые «плавные» переходы, неправильные громкости и НЕ будет применён зум 1.04+(-0.04) — водяной знак Veo останется в кадре. Эталон — живой Тесей в CapCut + `template_theseus.py`, не файлы `build_*.py`.
   - **Без `karaoke_<миф>.py`** в драфте останутся обычные статичные субтитры из шага 10.1 (мелкие, по центру внизу, не капсом). Это плейсхолдер.
   - При изменении набора сцен/шотов — синхронно править `scene_structure_<миф>.py` (для build) и `SCENE_LAYOUT` (для enrich), иначе индексы в треке разъедутся.
   - Утилита `pyCapCut/_inspect_theseus_live.py` печатает все актуальные переходы/эффекты/громкости/шрифты/зум из живого Тесея — использовать при подборе значений для нового мифа или для перепроверки `template_theseus.py` после ручных правок.

   После шагов 10.1–10.3: открыть CapCut → Drafts → выбрать проект → при необходимости подправить мышкой → Export в `content/<Название мифа>/final/<Название мифа>.mp4`.

## Правила контента

- Персонажи — антропоморфные бипедальные коты (не люди, не реалистичные кошки)
- Стиль — pixel art / 16-bit retro game style
- Ударения в текстах только на именах собственных (Ика́р, Деда́л)
- Видео-промпты обязательно содержат негатив: "No speech, no dialogue, no talking, no voices, no mouth movement, no music"
