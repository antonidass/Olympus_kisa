# pyCapCut — сборка CapCut-драфтов из Python

Главный инструмент монтажа канала. Идея: вместо ручного монтажа в CapCut
с нуля мы программно генерируем **CapCut-проект (draft)** через
[pyCapCut](https://github.com/GuanYixuan/pyCapCut) — раскладываем сцены,
озвучку, музыку, субтитры, переходы и SFX по точному эталону. Открываешь
готовый драфт в CapCut, при желании правишь руками и рендеришь финал
оттуда.

---

## Эталон монтажа: Тесей и Минотавр

С 2026-04-27 эталоном считается живой CapCut-драфт **«Тесей и Минотавр»**.
Все эталонные значения (зум, шрифты, переходы, эффекты, громкости, SFX,
fade-out) зафиксированы в [`template_theseus.py`](template_theseus.py).
Скрипты `enrich_<миф>.py` и `karaoke_<миф>.py` для будущих мифов
**обязаны импортировать оттуда**, не дублировать константы.

Утилита [`_inspect_theseus_live.py`](_inspect_theseus_live.py) печатает
свежие значения из живого драфта Тесея — запускать после ручных правок,
чтобы перепроверить и при необходимости обновить `template_theseus.py`.

См. также раздел `## Эталон монтажа канала: Тесей и Минотавр` в
корневом [`CONTEXT.md`](../CONTEXT.md).

## Установка

1. Python 3.10+ (у тебя уже стоит 3.13).
2. Установить зависимости:
   ```bash
   pip install -r pyCapCut/requirements.txt
   ```
3. Установить CapCut (десктоп). Проверено с CapCut International.
4. **ffmpeg в PATH** — нужен для `swap_audio_v2_*.py` и `karaoke_*.py`.

## Пайплайн сборки нового мифа

Три шага в строгом порядке: build → enrich → karaoke. Подробности —
в корневом [`README.md`](../README.md), раздел «9. Монтаж».

### 9.1 — `build_<миф>.py`
Базовая раскладка таймлайна:
- **Видео-дорожка**: scene_NN_M.mp4, делятся по сценам поровну внутри
  сцены, звук исходных mp4 глушится.
- **Озвучка**: sentence_NN.mp3 из `content/<миф>/voiceover/audio/approved_sentences/`.
- **Музыка**: `assets/music/Dorian_Concept_-_Hide_CS01_Version_(SkySound.cc).mp3`.
- **SFX**: whoosh на каждом slide-переходе из `assets/audio/WHOOSH.mp3`.
- **Субтитры-плейсхолдер**: текст сцены внизу кадра (потом `karaoke_*.py`
  заменит на честное караоке через whisper).
- **Базовые переходы**: «плавный» по умолчанию (потом `enrich_*.py`
  перебьёт по эталону Тесея).

### 9.2 — `enrich_<миф>.py`
Применяет эталон Тесея:
- Зум 1.04 + transform.y −0.04 на всех видео-сегментах (убирает водяной
  знак Veo).
- Чистит старые «плавные» переходы, ставит выбранные из
  `TRANSITIONS_LIBRARY` под драматургию.
- Расставляет видео-эффекты («Ожоги на пленке» на сцене 01,
  «Финальный круг» на финальной).
- Громкости: voice 1.0, video 0.34, music 0.1954, sfx 1.0.
- Fade-out 2.9s на музыкальной дорожке в конце.
- Whoosh-SFX на каждом стыке сцен (3 файла чередуются).

### 9.3 — `karaoke_<миф>.py`
Караоке-субтитры через whisper:
- Транскрибирует sentence_NN.mp3 через whisper-medium с word_timestamps.
- Выравнивает с эталонным текстом из `voiceover/texts/sentence_NNN.txt`.
- Удаляет статичные субтитры из шага 9.1.
- Добавляет text-track интро (Forum 20 bold, накопительные шаги) +
  text-track караоке (Philosopher 12 bold, КАПС, белый с обводкой).

## Где CapCut хранит драфты

Скрипты сами ищут в стандартных местах:

- **CapCut International (EN):**
  `%LOCALAPPDATA%\CapCut\User Data\Projects\com.lveditor.draft`
- **剪映 (CapCut China):**
  `%APPDATA%\JianyingPro\User Data\Projects\com.lveditor.draft`

Если у тебя портативная версия или CapCut переустановлен в нестандартное
место — передай путь через `--drafts`.

```

├── template_theseus.py           ★ ЭТАЛОН: зум, шрифты, переходы, эффекты, громкости, SFX

├── _karaoke_cache_<миф>.json     кэш whisper-транскрипций
└── README.md                     этот файл
```

## Известные ограничения

1. **Имена переходов в pyCapCut — китайские** (`淡化`, `闪白`, `放大`…).
   В разных версиях CapCut набор может отличаться. Скрипты используют
   `effect_id` напрямую (см. `template_theseus.TRANSITIONS_LIBRARY`),
   так что имена не критичны.
2. **Зум внутри клипа** (медленный pan/zoom внутри сцены) — в текущих
   скриптах не делается. Если нужно — добавляешь в CapCut вручную через
   ключевые кадры на клипе.
